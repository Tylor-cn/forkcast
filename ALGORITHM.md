# 算法设计

> Forkcast 推荐引擎的完整设计文档、验证数据与收敛性分析。

Forkcast 的推荐算法基于四项经过同行评审的认知心理学理论：

| 理论 | 应用 |
|:---|:---|
| **Elimination by Aspects** (Tversky, 1972) | 非补偿性两阶段决策模型——先排除再精选 |
| **Negativity Bias** (Baumeister et al., 2001) | 拒绝反馈权重加倍惩罚，快速收敛 |
| **Information Foraging** (Pirolli & Card, 1999) | 三卡斑块策略最大化信息密度 |
| **Affective Forecasting** (Wilson & Gilbert, 2003) | 情绪/时间/季节的一致性偏好建模 |

---

## 信号体系

三层非对称反馈信号，与认知心理学理论严格对齐：

| 操作 | 信号强度 | 短期权重变化 | 含义 |
|------|---------|-------------|------|
| 换一批 | 负向信号 | ×0.35 → ×0.22 (连续递增) | "这 3 个方向不对" |
| 备选 ⭐ | 弱正向信号 | ×1.2 | "可以考虑" |
| 选定 ❤️ | 强正向信号 | ×1.5 | "就吃这个" |

> 用户带着偏好来，系统通过排除式决策在 3~5 轮内快速收敛。纯浏览场景请使用菜单页。

### 菜系空间探索进度

以 470 道菜品、17 菜系、中午 12 点初始状态追踪 10 轮算法执行：

![菜系空间探索进度](./public/chart-exploration.svg)

```
1~5 轮 (首次扫描):  多样性机制确保每轮探索 3 个新菜系
                    → 5 轮覆盖 15/17 菜系 (88%)
6~10轮 (重复命中):  菜系用尽，已惩罚菜系重新出现
                    → 权重二次下降 → 逐轮触底 0.2
```

## 关键设计自查

- **不会出现同菜系扎堆** — 三槽策略（安全/熟悉/新奇）+ 菜系多样性权重 0.5
- **换一批不会过度惩罚单菜系** — 每次 3 道菜分散在不同菜系，惩罚均匀分布
- **连续拒绝升级方向正确** — 采用除法公式 `×0.35 / escalation`，连续拒绝惩罚递增
- **`season-all` 标签有效** — 364 道四季菜在任意季节均可获得上下文加成
- **排除优先级高于选择** — reject 惩罚力 (×0.22~0.35) 远大于 pick 奖励 (×1.5)

## 技术架构

```
┌─────────────────────────────────────────────┐
│              推荐引擎 Engine                 │
├─────────────────────────────────────────────┤
│  标签权重评分 + 多样性算法                    │
│  时间衰减 + 上下文加成                         │
│  非对称反馈 (拒绝惩罚 >> 选择奖励)              │
│  三槽策略 (安全/熟悉/新奇)                     │
├─────────────────────────────────────────────┤
│              存储 & 监控                      │
├─────────────────────────────────────────────┤
│  Zustand (5 Slice 模块化状态)                 │
│  LocalStorage Persist (纯客户端)              │
│  算法调参常量 → src/store/constants.ts        │
│  图表生成脚本 → scripts/generate-charts.cjs   │
└─────────────────────────────────────────────┘
```

### 调参指南

所有算法超参集中在 `src/store/constants.ts`，修改后运行 `node scripts/generate-charts.cjs` 重新生成验证图表。

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `REJECT_SHORT_PENALTY` | 0.35 | 单次拒绝短期惩罚乘数 |
| `CUISINE_REJECT_SHORT_PENALTY` | 0.50 | 菜系级别拒绝惩罚乘数 |
| `PICK_SHORT_BOOST` | 1.50 | 选定短期奖励乘数 |
| `BACKUP_SHORT_BOOST` | 1.20 | 备选短期奖励乘数 |
| `CONSECUTIVE_REJECT_FACTOR` | 0.12 | 连续拒绝升级系数 |
| `CONTEXT_BONUS_TIME` | 1.30 | 时段上下文加成 |
| `CONTEXT_BONUS_SEASON` | 1.15 | 季节上下文加成 |
